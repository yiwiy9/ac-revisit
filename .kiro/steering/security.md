# Security Standards

このプロジェクトのセキュリティ方針は、userscript を最小権限・ブラウザ内完結で動かし、不要なデータ収集や外部境界を増やさないことを基本とする。

## Philosophy

- **最小権限**: userscript grant は必要最小限に限定する。
- **閉じた実行境界**: 外部 API や外部サーバーを追加せず、状態はブラウザ内で完結させる。
- **fail-closed**: DOM や保存状態を安全に確認できない場合は、機能を静かに停止する。
- **最小データ保持**: 復習機能に不要な属性や利用者情報を保存しない。

## Runtime Boundary

- 実行対象は AtCoder ドメイン上の userscript に限定する。
- 配布物は単一ファイルとし、外部 JavaScript の `@require` 依存を追加しない。
- 新しいネットワーク通信、追跡、分析タグ、外部同期を導入しない。
- ブラウザ外のサービスに依存する機能を追加する場合は、MVP の前提変更として明示的に扱う。

## Permissions

- Tampermonkey grant は `GM_getValue` と `GM_setValue` を基本線とする。
- 新しい `@grant` を追加する場合は、「その権限がないと成立しないか」を先に検証する。
- 追加権限は convenience ではなく必須用途に限定し、権限追加に応じて steering も更新する。

## Stored Data

- 永続化するのは、問題識別子、表示用タイトル、登録日、および当日の提案状態に必要な最小情報だけとする。
- アカウント分離用の識別子、トークン、認証情報、個人情報は保存しない。
- 保存値に未知の追加属性があっても、それを信頼せず正規化時に落とす。
- 保存形式や論理整合性が崩れている場合は、部分復旧せず canonical な空状態へフォールバックする。

## DOM and Input Safety

- AtCoder DOM は信頼済み UI 文脈として扱うが、解決できない構造を無理に推測しない。
- 画面から読んだ文字列は、必要な識別子・表示文言としてのみ扱い、保存可能な最小形式へ絞る。
- 新しい入力面を追加する場合は、保存前に型と制約を明示し、任意オブジェクトをそのまま保存しない。

## Diagnostics and Logging

- 診断は `code`, `component`, `operation` の最小構造に限定する。
- 診断に、保存済み全文、DOM 全文、秘密値、将来追加される認証情報を載せない。
- 通常利用では恒常ログ出力を行わず、開発時に注入された sink へだけ流す。
- 利用者向け通知より、内部追跡可能性を優先するが、情報量は最小限に抑える。

## Dependency and Build Posture

- 配布物には必要コードを内包し、実行時に外部スクリプトを読み込まない。
- build metadata はテストで固定し、`@match` `@grant` `@require` の drift を検知する。
- セキュリティ上の境界を変える変更は、build script と contract test の両方を更新して一貫させる。

## Change Triggers

- 次の変更はセキュリティ方針の更新対象とする:
  - 新しい `@grant` の追加
  - 外部通信の導入
  - 保存データ項目の追加
  - 利用者入力欄や認証連携の追加
- 上記に該当しない変更は、既存の最小権限・最小保存の範囲に従う限り steering 更新を不要とする。

---
_具体的な秘密情報ではなく、権限・保存・通信の境界ルールを固定する_
