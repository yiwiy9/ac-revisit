# Product Overview

`ac-revisit` は AtCoder 上で動作する Tampermonkey 向け userscript であり、復習したい問題をブラウザ内だけで静かに管理し、14 日経過後に「今日の一問」として 1 問ずつ提案する。

## Core Capabilities

- AtCoder の問題ページと提出詳細ページで、現在の問題を復習対象へ追加・解除できる。
- ログイン済みユーザーにだけ、ユーザーメニュー内の常設導線から当日の状態確認を提供する。
- 当日最初の提案処理でのみ、条件を満たした候補から 1 問を選定する。
- 「完了」と「もう一問」の最小操作だけで、次の復習サイクルへ戻せる。

## Target Use Cases

- 過去に解いた問題を、義務感を増やさずに少しずつ復習したい AtCoder 利用者。
- 問題一覧や統計ではなく、その日に着手する 1 問だけを素早く確認したい利用者。
- 外部サービス連携なしで、同じブラウザプロファイル内に状態を閉じ込めたい利用者。

## Value Proposition

- 復習対象の保存情報を最小限に保ち、運用コストを低くしている。
- 提案は常に 1 問のみで、一覧管理や通知過多を避ける。
- AtCoder DOM に対して fail-closed で振る舞い、対応できない画面では静かに停止する。
- 外部 API やサーバーに依存せず、userscript 単体で完結する。

## Product Boundaries

- MVP では、自動 AC 検知、一覧画面、統計、メモ、タグ、ストリーク、動的間隔調整は扱わない。
- 完了判定はシステム推定ではなく、利用者の明示操作を唯一の根拠とする。
- 複数タブ間の厳密整合より、最後に成功した保存結果を採用する単純さを優先する。
